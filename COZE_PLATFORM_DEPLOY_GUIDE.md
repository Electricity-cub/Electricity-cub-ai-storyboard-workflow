## 🚀 开始部署

### 第1步：登录Coze平台

#### 1.1 打开Coze网站
在浏览器输入：**https://www.coze.com/**

#### 1.2 登录账户
- 点击右上角"登录"
- 选择登录方式（手机号、微信、邮箱等）
- 完成登录

#### 1.3 进入工作台
登录后，你会看到Coze工作台页面，包含：
- 左侧导航栏
- 中间的应用列表
- 右上角的创建应用按钮

---

### 第2步：创建工作流应用

#### 2.1 找到创建按钮
在页面中间区域，找到"**+ 创建应用**"按钮
- 位置：通常在页面顶部或应用列表上方
- 样式：蓝色或紫色的按钮，带"+"图标

#### 2.2 点击创建应用
点击"**+ 创建应用**"按钮

#### 2.3 选择应用类型
弹出的对话框中会显示多种类型：
- 🤖 **Bot** - 智能对话机器人
- 🔄 **工作流** - 自动化流程
- 📱 **应用** - 独立应用
- 🔌 **插件** - 功能扩展

**点击"工作流"卡片或选项**

#### 2.4 填写应用信息

**应用名称（必填）：**
```
AI分镜师工作流
```

**应用描述（选填）：**
```
将剧本转化为Seedance 2.0视频提示词的自动化工作流
```

**应用图标（选填）：**
- 可以跳过，使用默认图标
- 或上传自定义图标

#### 2.5 创建应用
- 点击"**创建**"或"**确认**"按钮
- 等待应用创建成功（通常几秒钟）

#### 2.6 查看应用界面
创建成功后，你会看到：
- 应用名称在顶部显示
- 左侧：节点面板（各种节点类型）
- 中间：工作流编辑画布
- 右侧：属性配置面板

---

### 第3步：关联GitHub仓库

#### 3.1 进入应用设置
- 点击页面右上角的"**设置**"或"**配置**"图标
- 通常是齿轮图标⚙️

#### 3.2 找到Git集成选项
在设置页面中，查找以下选项之一：
- "代码管理"
- "Git集成"
- "Git仓库"
- "关联仓库"

**位置提示：**
- 可能在左侧菜单中
- 可能在页面中部或底部

#### 3.3 点击关联仓库
找到后，点击"**关联仓库**"或"**添加仓库**"按钮

#### 3.4 选择Git平台
弹出的对话框中，选择：
- **GitHub** ← 选这个
- GitLab（如果你用的是GitLab）
- Gitee（如果你用的是Gitee）

#### 3.5 授权GitHub
点击GitHub后，可能会弹出GitHub授权页面：
- 点击"**Authorize**"或"**授权**"
- 允许Coze访问你的GitHub仓库

#### 3.6 选择仓库
授权后，会显示你的GitHub仓库列表：
- 找到：`Electricity-cub-ai-storyboard-workflow`
- 点击选择这个仓库

#### 3.7 选择分支
选择分支：
- **main** ← 选这个

#### 3.8 确认关联
点击"**确认**"或"**关联**"按钮

#### 3.9 验证关联成功
关联成功后，你会看到：
- 仓库名称：`Electricity-cub-ai-storyboard-workflow`
- 分支：`main`
- 最后提交时间
- 提交信息

---

### 第4步：配置部署参数

#### 4.1 找到部署配置
在应用设置页面，查找：
- "部署配置"
- "构建配置"
- "运行配置"

#### 4.2 配置构建命令

**找到"构建命令"或"Build Command"输入框**

输入：
```bash
bash scripts/setup.sh
```

**这个命令的作用：**
- 安装项目依赖（Python包）
- 配置环境
- 准备运行环境

#### 4.3 配置运行命令

**找到"运行命令"或"Run Command"输入框**

输入：
```bash
bash scripts/http_run.sh -p 5000
```

**这个命令的作用：**
- 启动HTTP服务
- 监听5000端口
- 提供API接口

#### 4.4 配置环境变量（可选）

**找到"环境变量"或"Environment Variables"**

可以添加以下环境变量：
```
PORT=5000
LOG_LEVEL=INFO
```

#### 4.5 配置入口文件（如果有）

**找到"入口文件"或"Entry Point"**

确保是：
```
src/main.py
```

#### 4.6 保存配置
点击"**保存**"或"**确认**"按钮

---

### 第5步：触发部署

#### 5.1 打开部署页面
- 在应用界面，找到"**部署**"或"**Deploy**"按钮
- 通常在右上角或顶部导航栏
- 点击进入部署页面

#### 5.2 选择部署分支
在部署页面，找到"**分支**"或"**Branch**"选择框
- 选择：**main**

#### 5.3 开始部署
点击"**开始部署**"或"**Deploy**"按钮

#### 5.4 查看部署进度
部署开始后，会显示：
- 部署状态：**部署中** / **Deploying**
- 实时日志输出
- 进度条或步骤指示

**部署过程包含：**
1. 拉取代码（从GitHub）
2. 安装依赖（Python包）
3. 构建项目
4. 启动服务
5. 健康检查

#### 5.5 等待部署完成
预计时间：**3-5分钟**

**你可以看到：**
- 每个步骤的日志
- 进度百分比
- 预计剩余时间

#### 5.6 部署成功
部署完成后，会显示：
- 状态：**部署成功** / **Success**
- 绿色对勾✅
- 应用访问地址（URL）

---

### 第6步：获取访问地址

#### 6.1 查看应用地址
部署成功后，在部署页面或应用首页，你会看到：

**应用地址（URL）示例：**
```
https://your-app-id.coze.run
```

或

```
https://your-app-id.coze.cn
```

#### 6.2 复制访问地址
点击地址旁边的"**复制**"按钮

#### 6.3 测试访问
在浏览器中打开这个地址

---

### 第7步：验证部署

#### 7.1 健康检查
在浏览器访问：
```
https://your-app-id.coze.run/health
```

**期望结果：**
```json
{
  "status": "healthy",
  "service": "AI Storyboard Workflow"
}
```

#### 7.2 API测试
在浏览器或使用工具（如Postman），发送POST请求：

**URL：**
```
https://your-app-id.coze.run/api/v1/run
```

**请求方法：** POST

**请求头：**
```
Content-Type: application/json
```

**请求体：**
```json
{
  "script_content": "小明和小红在公园相遇，一见钟情。",
  "episode_number": "ep01",
  "visual_style": "写实",
  "project_type": "国内短剧"
}
```

**期望结果：**
返回一个JSON对象，包含：
- director_notes：导演讲戏本
- character_prompts：人物提示词
- scene_prompts：场景提示词
- storyboard：分镜头脚本
- seedance_prompts：视频提示词

## ⚠️ 常见问题

### Q1: 找不到"关联仓库"选项怎么办？
**A:** 检查以下几点：
- 确认已登录Coze
- 确认创建了工作流类型的应用
- 查看应用设置的所有选项卡
- 查看帮助文档或联系Coze客服

---

### Q2: 关联仓库时授权失败怎么办？
**A:** 可能的原因：
- GitHub账户权限问题
- 网络连接问题
- Token已过期

**解决方法：**
- 重新生成GitHub Token
- 检查网络连接
- 刷新页面重试

---

### Q3: 部署时提示"构建失败"怎么办？
**A:** 查看部署日志，检查：
- pyproject.toml文件是否正确
- 依赖是否都能正常安装
- scripts/setup.sh脚本是否存在

---

### Q4: 部署时提示"启动失败"怎么办？
**A:** 查看部署日志，检查：
- src/main.py文件是否存在
- scripts/http_run.sh脚本是否正确
- 端口5000是否被占用

---

### Q5: 部署成功后无法访问怎么办？
**A:** 检查：
- 应用地址是否正确复制
- 网络连接是否正常
- 浏览器是否能正常访问
- 防火墙是否阻止访问

---

### Q6: API调用返回错误怎么办？
**A:** 检查：
- 请求URL是否正确
- 请求方法是否为POST
- 请求头是否包含`Content-Type: application/json`
- 请求体格式是否正确
